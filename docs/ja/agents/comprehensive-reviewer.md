# comprehensive-reviewer

**フェーズ:** 7 — 包括的レビュー

## 役割

タスク横断の総合レビュアー。すべての実装タスクを横断的な観点で調査します。

## 入力

- すべての `impl-{N}.md` と `review-{N}.md` ファイル
- フル機能diff（`git diff main...HEAD`）

## 出力

- `comprehensive-review.md` — 判定（CLEAN または IMPROVED）

## 制約

- `bugfix`、`investigation`、`docs` タスクタイプではスキップ
- 工数 S（`light` テンプレート）ではスキップ

## レビュー内容

- すべての変更ファイルにわたる**命名の一貫性**
- 別タスクで導入された**コードの重複**
- コンポーネント間の**インターフェースの整合性**
- **完全性** — すべての設計要素が実装されているか
- **テストカバレッジのギャップ** — 欠落しているエッジケース

## 判定

| 判定 | 意味 |
| --- | --- |
| **CLEAN** | 問題なし |
| **IMPROVED** | 問題を発見し直接修正 |

他のレビュアーと異なり、このエージェントは修正のために差し戻すのではなく、**問題を直接修正**します。
